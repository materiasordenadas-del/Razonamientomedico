# Reglas del proyecto H-CR

- Usar HTML, CSS y JavaScript puro.
- No usar React, Node, npm, Vite, TypeScript, backend ni base de datos sin aprobacion explicita.
- app/ contiene diseno y funcionamiento general.
- cases/ contiene solamente datos clinicos de cada caso.
- No modificar caso-1.html, index.html ni legacy/ sin autorizacion explicita.
- No generar imagenes clinicas falsas.
- No anadir, quitar ni cambiar datos clinicos sin autorizacion.
- Antes de cada cambio, explicar que archivos se modificaran.
- Hacer cambios pequenos y delimitados.
- Despues de cada cambio, abrir la vista previa y comprobar que funciona.
- Informar que se probo, que funciono y que fallo.
- No declarar una tarea terminada sin probarla visualmente.
- No reorganizar archivos ni reescribir modulos completos salvo que se solicite expresamente.
